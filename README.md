# mmmSIM

A high-performance MATLAB framework for 2-beam 3D Structured Illumination Microscopy (SIM) reconstruction. `mmmSIM` is designed to handle large-scale volumetric imaging data by leveraging multi-GPU parallelization and batched processing workflows.
This is adapted from Mats Gustafsson & Lin Shao's 3-beam SIM reconstruction software, with CUDA acceleration (https://github.com/scopetools/cudasirecon). 
Algorithm as described in Gustafsson et al (2008) Biophys. J. 94(12): 4957–4970. doi: 10.1529/biophysj.107.120345

## Features
* **2-Beam 3D SIM Reconstruction**: Core reconstruction algorithms tailored for processing complex volumetric imaging data (`sim_recon_2beam_3d_core.m`).
* **Hardware Acceleration**: Multi-GPU, batched, and parallelized execution scripts to distribute reconstruction loads efficiently (`runSIMrecon_folder_multigpu_channels_fixed.m`, `sim_recon_2beam_3d_unified_parallel_batched.m`).
* **Analytic Parameter Refitting**: Dynamic estimation and refitting of SIM parameters (`refit_sim_params_2beam_3d.m`) with thresholded parallel execution.
* **Canonical OTF Generation**: Generate 3D Optical Transfer Functions specifically tailored for 2-beam optical systems (`make_otf_2beam_3d_canonical.m`).

## Prerequisites
* **MATLAB** with the **Parallel Computing Toolbox**.
* **CUDA-enabled NVIDIA GPUs**: The batched parallelization strategies are designed to scale smoothly on modern compute hardware, allowing you to maximize tensor core performance and reconstruction efficiency on architectures like the NVIDIA H100, H200, or B200.

## Usage
1. **Prepare Data**: Ensure your raw SIM acquisitions are appropriately structured in the target input folders. 
2. **Configure Parameters**: If you need to generate a new OTF for your specific optical configuration, run `make_otf_2beam_3d_canonical.m` before beginning reconstruction. 
3. **Execute Pipeline**: 
   Launch `runSIMrecon_folder_multigpu_channels_fixed.m` or `runSIMrecon_refit_analytic_thresholded_parallel_batched.m` to initiate the processing pipeline. The scripts will automatically handle batching and distribute the workload across the available GPUs.

## Repository Structure
* `make_otf_2beam_3d_canonical.m`: Generates the canonical OTF for 2-beam 3D SIM.
* `refit_sim_params_2beam_3d.m`: Algorithm for analytic refitting of SIM parameters.
* `runSIMrecon_folder_multigpu_channels_fixed.m`: Main wrapper to run reconstructions on a directory using multiple GPUs.
* `runSIMrecon_refit_analytic_thresholded_parallel_batched.m`: Advanced runner utilizing batched analytic refitting.
* `sim_recon_2beam_3d_core.m`: Core 3D reconstruction mathematical routines.
* `sim_recon_2beam_3d_core_batched_shared.m`: Batched, shared-memory variant of the core reconstruction.
* `sim_recon_2beam_3d_unified_parallel_batched.m`: Unified parallel batched reconstruction routine.
